# 2. Business scenario

You are working as a **junior data engineer** for a **London transport analytics team**.

The organization receives operational transport data from multiple internal systems and wants to prepare it for reporting and analysis.

Management wants answers to questions such as:

- Which stations are the busiest?
- Which lines carry the highest passenger volume?
- Which journeys are delayed most often?
- Which transport modes show the most activity?
- Which areas of London generate the most transport traffic?

These questions are the same as Day 1.

What changes now is the way you process the data.

Instead of solving the problem through local ETL and ELT with PostgreSQL, you will now use **Spark** to process the same business problem in a more scalable data engineering style.
